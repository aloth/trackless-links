# Trackless Links 常见问题

**常见问题解答与用户指南**

欢迎！本指南将帮助你充分利用 Trackless Links，并解答常见问题。

---

## 背景：为什么干净的 URL 很重要

1989年，蒂姆·伯纳斯-李发明万维网时，URL 被设计成简单的地址：一种在服务器上找到文档的方式。像 `shop.com/shoes` 这样的 URL 清楚地告诉你要去哪里。

如今，URL 已经变成了监控工具。

### 你点击背后隐藏的经济

每次你点击电子邮件中的链接、点击社交媒体上的帖子或跟随搜索结果时，额外的代码就会被附加到 URL 上。这些追踪参数将一个简单的地址变成了关于你的详细报告：

```
shop.com/shoes?utm_source=newsletter&utm_campaign=spring_sale&utm_medium=email&fbclid=IwAR3x...&mc_eid=abc123
```

这一个 URL 告诉多家公司：
- 你来自电子邮件订阅（而不是直接访问网站）
- 你对"春季促销"活动做出了反应
- Facebook 知道你点击了这个特定链接
- 电子邮件营销平台记录了你唯一的订阅者 ID

这些都不会影响你看到的页面内容，只会影响公司了解到的关于你的信息。

### 追踪参数如何工作

URL 中的问号（?）将页面地址与其参数分开。问号之后的所有内容都是元数据，公司们已经将其标准化为一个监控系统：

| 参数 | 使用者 | 暴露的信息 |
|------|--------|-----------|
| `utm_source` | 各类营销人员 | 哪个网站或平台引导你来的 |
| `utm_campaign` | 各类营销人员 | 你点击了哪个具体广告或邮件 |
| `fbclid` | Facebook/Meta | 你的 Facebook 用户 ID 和点击行为 |
| `gclid` | Google | 你的 Google 广告档案 |
| `mc_eid` | Mailchimp | 你的邮件订阅者身份 |
| `ttclid` | TikTok | 你的 TikTok 广告档案 |
| `msclkid` | Microsoft/Bing | 你的 Microsoft 广告档案 |

当你与朋友分享链接时，往往也会分享这些参数，这让公司不仅能追踪你，还能追踪每个点击你转发链接的人。

### 隐私的代价

这种追踪是无形的，但后果是真实的：

**跨站画像**：Facebook 和 Google 等公司可以看到你在数百万网站上的活动，建立你的兴趣、购买习惯的详细档案。

**价格歧视**：一些零售商使用追踪数据，根据用户感知到的支付意愿向不同用户显示不同价格。

**过滤气泡**：广告平台使用你的点击历史来决定向你展示什么内容，可能会限制你接触多元化观点的机会。

**数据泄露**：每个存储你追踪数据的公司都是潜在的泄露目标。收集的数据越少，可被窃取的数据就越少。

### Trackless Links 做什么

Trackless Links 在 URL 加载之前拦截它们，并剥离追踪参数。清理后的 URL 仍然带你到同一个页面，但不会广播你的数字指纹。

清理前：`amazon.cn/dp/1119737281?tag=influencer-20&ref=pd_sl_abc&utm_source=instagram`

清理后：`amazon.cn/dp/1119737281`

这对 Safari 中的每个链接都会自动进行，将 URL 恢复到它们本来的样子：简单的地址，仅此而已。

## 入门

### 如何启用 Safari 扩展？

安装应用后：

1. 在你的设备上打开 **Safari**
2. 点击地址栏中的 **AA** 图标（Mac 上是**扩展**按钮）
3. 点击**管理扩展**
4. 找到 **Trackless Links** 并将其**打开**
5. 授予必要的权限

扩展现在已激活，将自动开始清理 URL。

### 为什么扩展不工作？

确保你已经：
- 在 Safari 设置中启用了扩展（见上文）
- 在提示时授予了所需权限
- 扩展需要访问网页内容的权限才能工作

在 Mac 上，你可能还需要检查**系统设置 > 隐私与安全性 > 扩展**并启用 Trackless Links。

### 开始使用需要配置什么吗？

不需要。Trackless Links 开箱即用，默认设置合理：
- 追踪参数会自动删除
- 20 多个预定义重定向可随时使用（你可以启用需要的）
- 所有调整都是可选启用的，只启用你需要的

## 隐私与追踪过滤器

### 它会删除哪些追踪参数？

Trackless Links 删除常见的追踪代码，如：
- `utm_source`、`utm_medium`、`utm_campaign`（营销归因）
- `fbclid`（Facebook 点击标识符）
- `gclid`（Google 点击标识符）
- `msclkid`（Microsoft/Bing 点击标识符）
- 以及更多

你可以在**过滤器**标签中查看完整列表，并添加自己的自定义参数。

### 如何添加自定义追踪参数？

1. 打开 Trackless Links 应用
2. 转到**过滤器**标签
3. 点击 **+** 按钮
4. 输入参数名称（例如 `my_tracker`）
5. 点击**保存**

下次访问包含该参数的 URL 时，它将被自动删除。

### 可以临时禁用追踪删除吗？

可以。在**过滤器**标签中，你可以单独开启或关闭各个过滤器。你也可以通过关闭所有过滤器来禁用所有追踪删除。

## 重定向

### 什么是重定向，为什么要使用它们？

许多热门网站会收集用户的大量数据。重定向让你可以自动跳转到提供相同内容但不进行监控的隐私友好替代方案。

**注重隐私的前端**：Nitter（用于 Twitter/X）、Invidious（用于 YouTube）和 Libreddit（用于 Reddit）等项目可以显示主要平台的内容，而无需登录或追踪你的行为。你获得内容，他们得不到你的数据。

**界面偏好**：Reddit 的 old.reddit.com 提供许多用户偏好的经典界面。一些新闻网站有更简洁的移动版本。重定向让你始终访问你喜欢的版本。

**地区路由**：自动将国际域名重定向到你的本地版本，反之亦然。

### 如何启用预定义重定向？

1. 打开应用并转到**重定向**
2. 浏览预定义规则列表
3. 切换任意规则旁边的开关以启用它
4. 重定向立即生效，无需重启

### 如何创建自定义重定向？

1. 转到**重定向**标签
2. 点击 **+** 按钮
3. 输入：
   - **模式**：你要从哪个 URL 重定向（例如 `reddit.com`）
   - **替换为**：重定向到哪里（例如 `old.reddit.com`）
   - **标题**（可选）：这条规则的友好名称
4. 点击**保存**

你可以使用精确匹配或正则表达式（regex）进行高级模式匹配。

### 可以重新排列重定向规则吗？

可以。顺序很重要，因为第一个匹配的规则会生效。要重新排序：

1. 转到**重定向**
2. 长按并拖动规则来重新排列
3. 你的优先级会自动保存

### 我的重定向不工作，是什么问题？

检查这些常见问题：

- 规则是否**启用**？（开关已打开）
- 模式是否正确？（先用简单的精确匹配测试）
- 是否有其他规则先匹配了？（检查规则顺序）
- 你是在 Safari 中测试吗？（扩展只在 Safari 中工作，不支持其他浏览器）

## 调整

### 什么是调整？

现代网站经常限制你在自己浏览器中可以做的事情。它们禁用文本选择以防止复制，自动播放视频以提高参与度指标，并显示侵入性弹窗来阻止你离开。

调整帮你夺回控制权：

- 在阻止文本选择的网站上重新启用
- 阻止视频和音频自动播放
- 在任何网站上强制启用深色模式
- 移除"确定要离开此页面吗？"弹窗
- 更多功能

所有调整都是可选启用的，你可以控制启用哪些。

### 如何启用调整？

1. 转到**调整**标签
2. 浏览可用选项
3. 打开或关闭任何调整
4. 更改立即生效

### 为什么某个调整在特定网站上不起作用？

一些网站会主动对抗修改。如果调整不起作用：

- 该网站可能使用了激进的反修改技术
- 尝试禁用其他扩展以排除冲突
- 在 GitHub 上报告问题。我们一直在改进兼容性。

### "强制媒体控制"和"阻止自动播放"有什么区别？

- **强制媒体控制**：即使网站隐藏了控制按钮，也显示标准浏览器视频/音频控制
- **阻止自动播放**：防止视频/音频在加载页面时自动播放

你可以同时使用它们以获得最大控制。

## 存档搜索

### 什么是存档搜索？

网络不是永久的。页面会被删除，文章会被编辑，整个网站会消失。互联网档案馆的 Wayback Machine 和类似服务保存了网络的快照，创建了任意时间点存在内容的历史记录。

存档搜索让你即时访问这些保存的版本：

- 查找已删除的内容或查看页面更改前的样子
- 在网站暂时宕机时访问页面
- 研究网站及其声明如何随时间演变
- 验证可能被篡改的引用和来源

### 如何搜索存档？

**从 Safari：**
1. 点击 **AA** 图标或**扩展**按钮
2. 选择 **Trackless Links**
3. 点击你想要的存档服务（例如 Wayback Machine）
4. 当前页面将在该存档中被搜索

**从任何应用（共享表单）：**
1. 从任何应用分享链接
2. 选择**在存档中搜索**
3. 选择你喜欢的存档服务

### 可以添加更多存档服务吗？

可以。你可以添加自定义存档服务：

1. 转到**调整**，然后**存档搜索**，然后**配置服务**
2. 点击 **+** 添加服务
3. 输入存档服务的 URL 模式
4. 该服务现在可在弹出菜单中使用

### 应该启用"剥离查询参数"吗？

建议：是。此选项在搜索存档之前删除追踪代码和其他参数，通常能带来更好的匹配结果。只有当你需要搜索包含特定参数的 URL 时才禁用它。

## 共享表单集成

### 如何使用共享表单清理链接？

当你从其他应用分享链接时，追踪参数会一起跟随。共享表单扩展让你可以在分享前清理链接：

1. 在任何应用（Safari、Reddit、X 等）中点击**分享**按钮
2. 滚动找到**使用 Trackless Links 清理**
3. 清理后的 URL 会被复制到剪贴板

现在你可以分享一个干净的链接，不会透露你在哪里找到它，也不会追踪每个点击它的人。

### 可以从其他应用分享到存档吗？

可以。使用相同的共享表单：

1. 在任何链接上点击**分享**
2. 选择**在存档中搜索**
3. 选择你的存档服务
4. Safari 会打开存档搜索结果

## iCloud 同步

### iCloud 同步如何工作？

你的隐私设置与你同行。当你启用 iCloud 同步时，你的配置会通过个人 iCloud 账户在 iPhone、iPad 和 Mac 之间自动同步：

- **过滤器**（追踪参数）
- **重定向**（自定义规则和启用状态）
- **调整**（所有设置）
- **存档服务**（你配置的服务）

隐私统计数据（如已阻止的追踪器数量）保持设备特定，不会同步。

### 需要启用 iCloud 同步吗？

这是可选的。iCloud 同步默认禁用。要启用：

1. 在应用中转到**设置**
2. 找到 **iCloud 同步**
3. 将其**打开**

你的设置现在会在登录同一 iCloud 账户的所有设备间同步。

### 设备间的更改没有同步，是什么问题？

检查以下项目：

- 两台设备上的 iCloud 同步都**启用**了吗？
- 两台设备都登录了**同一个 iCloud 账户**吗？
- **系统设置**（Mac）或**设置**（iOS）中的 iCloud Drive 启用了吗？
- 两台设备都连接到互联网了吗？
- 尝试强制退出并重新打开应用

如果同步仍然不工作，尝试关闭 iCloud 同步然后重新打开。

### 可以不用 iCloud 使用 Trackless Links 吗？

可以。iCloud 同步完全是可选的。应用在每台设备上本地存储所有设置也能正常工作。

## 技巧与窍门

### 如何查看删除了哪些追踪参数？

查看主仪表板上的**统计数据**。它显示：
- 清理的 URL 总数
- 应用的重定向

### 可以导出我的设置吗？

设置存储在 iCloud（如果启用）或你的设备本地。所有数据都在你的控制下，从不发送到第三方服务器。

### Trackless Links 会减慢浏览速度吗？

不会。扩展非常轻量，在页面加载完成之前以毫秒级速度处理 URL。你不会注意到任何性能影响。

### 可以将 Trackless Links 与其他 Safari 扩展一起使用吗？

可以。Trackless Links 设计为与其他扩展协同工作。如果遇到冲突，尝试在 Safari 的扩展设置中调整执行顺序。

### Trackless Links 和 Trackless Links Pro 有什么区别？

| 功能 | Trackless Links | Trackless Links Pro |
|------|----------------|---------------------|
| iPhone 和 iPad | 是 | 是 |
| Mac 应用 | 否 | 是 |
| iCloud 同步 | 是 | 是 |
| 定价 | 一次性购买（仅 iOS） | 通用购买（全平台） |

Trackless Links Pro 是通用购买。购买一次即可在 iPhone、iPad 和 Mac 上使用。

## 故障排除

### 在 Safari 中找不到扩展

确保：
- 你已从 App Store 安装了应用
- 你至少打开过一次应用
- 系统设置中已启用 Safari 扩展

### 链接没有被清理

检查：
- Safari 中扩展是否**启用**？
- 你是否授予了必要的权限？
- 过滤器标签中是否有过滤器**启用**？

### 我的自定义重定向不工作

- 仔细检查模式语法（先尝试精确匹配）
- 确保规则已**启用**
- 检查是否有其他规则先匹配（规则顺序很重要）

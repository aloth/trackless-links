# Trackless Links FAQ

**よくある質問とユーザーガイド**

ようこそ。このガイドでは、Trackless Linksを最大限に活用するための情報と、よくある質問への回答をまとめています。

---

## 背景：なぜクリーンなURLが重要なのか

1989年、ティム・バーナーズ＝リーがWorld Wide Webを発明したとき、URLはシンプルな住所として設計されました。サーバー上のドキュメントを見つけるための手段です。`shop.com/shoes`のようなURLを見れば、どこに行くのか一目瞭然でした。

今日、URLは監視ツールになっています。

### クリックに隠された経済

メールのリンクをクリックしたり、SNSの投稿をタップしたり、検索結果をたどったりするたびに、URLには追加のコードが付け加えられます。これらのトラッキングパラメータは、シンプルな住所をあなたに関する詳細なレポートに変えてしまいます：

```
shop.com/shoes?utm_source=newsletter&utm_campaign=spring_sale&utm_medium=email&fbclid=IwAR3x...&mc_eid=abc123
```

このたった一つのURLが複数の企業に伝えること：
- あなたはメールマガジンから来た（ウェブサイトに直接アクセスしたのではない）
- 「春のセール」キャンペーンに反応した
- Facebookはあなたがこの特定のリンクをクリックしたことを知っている
- メールマーケティングプラットフォームがあなた固有の購読者IDを記録した

これらはどのページが表示されるかには影響しません。企業があなたについて何を知るかにだけ影響します。

### トラッキングパラメータの仕組み

URLの中の疑問符（?）は、ページアドレスとパラメータを分離します。疑問符の後ろはすべてメタデータであり、企業はこれを監視システムとして標準化しています：

| パラメータ | 使用者 | 明らかになる情報 |
|-----------|--------|------------------|
| `utm_source` | マーケター全般 | どのウェブサイトやプラットフォームから来たか |
| `utm_campaign` | マーケター全般 | どの広告やメールをクリックしたか |
| `fbclid` | Facebook/Meta | FacebookユーザーIDとクリック行動 |
| `gclid` | Google | Googleの広告プロファイル |
| `mc_eid` | Mailchimp | メール購読者としてのID |
| `ttclid` | TikTok | TikTokの広告プロファイル |
| `msclkid` | Microsoft/Bing | Microsoftの広告プロファイル |

友人にリンクを共有すると、これらのパラメータも一緒に共有されることが多く、企業はあなただけでなく、転送されたリンクをクリックしたすべての人を追跡できるようになります。

### プライバシーへの代償

このトラッキングは目に見えませんが、その影響は現実のものです：

**クロスサイトプロファイリング**：FacebookやGoogleのような企業は、何百万ものウェブサイトでのあなたの活動を見て、興味、購入履歴、習慣の詳細なプロファイルを構築しています。

**価格差別**：一部の小売業者はトラッキングデータを使用して、支払い意欲が高いと思われるユーザーに異なる価格を表示します。

**フィルターバブル**：広告プラットフォームはあなたのクリック履歴を使って、どのコンテンツを表示するかを決定します。これにより、多様な視点に触れる機会が制限される可能性があります。

**データ漏洩**：あなたのトラッキングデータを保存しているすべての企業が、潜在的な漏洩のターゲットです。収集されるデータが少なければ、盗まれるデータも少なくなります。

### Trackless Linksの役割

Trackless Linksは、URLが読み込まれる前にインターセプトし、トラッキングパラメータを削除します。クリーンなURLでも同じページに移動しますが、デジタルフィンガープリントを発信することはありません。

変換前：`amazon.co.jp/dp/1119737281?tag=influencer-20&ref=pd_sl_abc&utm_source=instagram`

変換後：`amazon.co.jp/dp/1119737281`

これはSafariのすべてのリンクで自動的に行われ、URLを本来の姿に戻します：シンプルな住所、それだけです。

## はじめに

### Safari拡張機能を有効にするには？

アプリをインストールした後：

1. デバイスで**Safari**を開く
2. アドレスバーの**AA**アイコン（Macでは**機能拡張**ボタン）をタップ
3. **機能拡張を管理**をタップ
4. **Trackless Links**を見つけて**オン**に切り替える
5. 必要な権限を許可する

拡張機能が有効になり、自動的にURLのクリーニングが始まります。

### 拡張機能が動作しないのはなぜ？

以下を確認してください：
- Safari設定で拡張機能を有効にした（上記参照）
- プロンプトが表示されたときに必要な権限を許可した
- 拡張機能が機能するにはウェブページコンテンツへのアクセス権限が必要

Macでは、**システム設定 > プライバシーとセキュリティ > 機能拡張**でTrackless Linksを有効にする必要がある場合もあります。

### 使い始めるのに設定は必要？

いいえ。Trackless Linksは適切なデフォルト設定ですぐに動作します：
- トラッキングパラメータは自動的に削除されます
- 20以上の定義済みリダイレクトがすぐに使えます（必要なものを有効にできます）
- すべてのTweaksはオプトインなので、必要なものだけを有効にしてください

## プライバシーとトラッキングフィルター

### どのトラッキングパラメータが削除される？

Trackless Linksは以下のような一般的なトラッキングコードを削除します：
- `utm_source`、`utm_medium`、`utm_campaign`（マーケティングアトリビューション）
- `fbclid`（Facebookクリック識別子）
- `gclid`（Googleクリック識別子）
- `msclkid`（Microsoft/Bingクリック識別子）
- その他多数

**フィルター**タブで完全なリストを確認し、独自のパラメータを追加できます。

### カスタムトラッキングパラメータを追加するには？

1. Trackless Linksアプリを開く
2. **フィルター**タブに移動
3. **+**ボタンをタップ
4. パラメータ名を入力（例：`my_tracker`）
5. **保存**をタップ

次にそのパラメータを含むURLにアクセスすると、自動的に削除されます。

### トラッキング削除を一時的に無効にできる？

はい。**フィルター**タブで個々のフィルターのオン・オフを切り替えられます。すべてのフィルターをオフにすることで、トラッキング削除全体を無効にすることもできます。

## リダイレクト

### リダイレクトとは？なぜ使うの？

多くの人気ウェブサイトは、ユーザーに関する膨大なデータを収集しています。リダイレクトを使えば、監視なしで同じコンテンツを提供するプライバシー重視の代替サービスに自動的に移動できます。

**プライバシー重視のフロントエンド**：Nitter（Twitter/X用）、Invidious（YouTube用）、Libreddit（Reddit用）などのプロジェクトは、ログインやトラッキングなしで主要プラットフォームのコンテンツを表示します。コンテンツは得られ、データは渡さない。

**インターフェース設定**：Redditのold.reddit.comは、多くのユーザーが好むクラシックインターフェースを提供します。一部のニュースサイトにはよりすっきりしたモバイル版があります。リダイレクトを使えば、常に好みのバージョンにアクセスできます。

**地域ルーティング**：国際ドメインをローカル版に自動的にリダイレクト、またはその逆。

### 定義済みリダイレクトを有効にするには？

1. アプリを開き、**リダイレクト**に移動
2. 定義済みルールのリストを閲覧
3. ルールの横にあるスイッチをオンにして有効化
4. リダイレクトはすぐに有効になり、再起動は不要

### カスタムリダイレクトを作成するには？

1. **リダイレクト**タブに移動
2. **+**ボタンをタップ
3. 入力：
   - **パターン**：リダイレクト元のURL（例：`reddit.com`）
   - **置換先**：リダイレクト先（例：`old.reddit.com`）
   - **タイトル**（オプション）：このルールの名前
4. **保存**をタップ

高度なパターンには完全一致または正規表現（regex）を使用できます。

### リダイレクトルールの順序を変更できる？

はい。最初にマッチしたルールが適用されるため、順序は重要です。順序を変更するには：

1. **リダイレクト**に移動
2. ルールを長押しして、ドラッグして並べ替え
3. 優先順位は自動的に保存されます

### リダイレクトが動作しない。何が問題？

以下の一般的な問題を確認してください：

- ルールは**有効**になっている？（トグルスイッチがオン）
- パターンは正しい？（まずシンプルな完全一致でテスト）
- 別のルールが先にマッチしていない？（ルールの順序を確認）
- Safariでテストしている？（拡張機能はSafariでのみ動作し、他のブラウザでは動作しません）

## Tweaks

### Tweaksとは？

現代のウェブサイトは、あなた自身のブラウザでできることを制限することがよくあります。コピーを防ぐためにテキスト選択を無効にし、エンゲージメント指標を上げるために動画を自動再生し、離脱を防ぐために侵入的なポップアップを表示します。

Tweaksはあなたにコントロールを取り戻します。Trackless Linksには、6つのカテゴリに整理された20以上の高度なブラウザ調整機能が含まれています。

すべてのTweaksはオプトインなので、どれを有効にするかはあなた次第です。

### Tweaksの完全なリスト

#### 閲覧と操作

| Tweak | 機能 |
|-------|------|
| **テキスト選択とコピーを許可** | ブロックしようとするサイトで選択、コピー、貼り付け、コンテキストメニューを再有効化。調査やメモ取りに不可欠。 |
| **終了ダイアログを無効化** | タブを離れたり閉じたりする際の「このページを離れますか？」プロンプトを防止。 |
| **強制新規タブを無効化** | サイトがリンクを新しいタブで強制的に開くのを停止（target="_blank"）。リンクはデフォルトで同じタブで開きます。 |
| **テキストフラグメントを無視** | URLからChrome形式の#:~:text=フラグメントを削除し、ハイライトされたテキストへの自動スクロールを防止。 |
| **ドラッグ＆ドロップを許可** | ブロックしているサイトでデフォルトのテキストドラッグとドラッグ＆ドロップ動作を復元。 |

#### Cookieバナー処理（実験的）

Trackless Linksは、3つのモードでインテリジェントなCookie同意管理を提供します：

| モード | 動作 |
|--------|------|
| **オフ** | Cookieバナーへの干渉なし |
| **非表示のみ** | 何もクリックせずにCookieバナーを視覚的に非表示 |
| **自動拒否** | 「拒否」または「必須のみ」ボタンを自動的にクリックし、残りのバナーを非表示 |

自動拒否機能は複数の言語をサポートし、Google、一般的な同意フレームワーク、多くの人気サイトで動作します。

#### メディアコントロール

| Tweak | 機能 |
|-------|------|
| **常に再生/一時停止コントロールを表示** | サイトが非表示にしていても、オーディオとビデオプレーヤーにネイティブブラウザコントロールを強制。 |
| **自動再生をブロック** | ページ読み込み時に動画と音声が自動的に再生されるのを停止。 |
| **デフォルト速度を適用** | すべてのメディアをお好みの再生速度（0.5倍〜3.0倍）に設定。 |
| **再生速度をロック** | サイトが選択した速度をリセットするのを防止。 |
| **ページ読み込み時にすべてのメディアを一時停止** | ページが読み込まれたときにすべてのオーディオとビデオ要素を一時停止。 |
| **ピクチャーインピクチャーを有効化** | 無効にしようとするサイトでPiP機能を復元。 |

#### プライバシーとクリーンアップ

| Tweak | 機能 |
|-------|------|
| **コピーしたURLをクリーン** | クリップボードにコピーしたリンクからトラッキングパラメータを自動的に削除。 |
| **一般的なリダイレクターをバイパス** | 短縮URL（t.co、bit.lyリンクなど）を展開し、直接目的地へ移動。 |

#### 追加Tweaks

| Tweak | 機能 |
|-------|------|
| **ページズーム制限を無効化** | モバイル非対応サイトでのズームイン・ズームアウト制限を解除。 |
| **ダークモードを強制** | インテリジェントな色反転を適用し、任意のウェブサイトにダークテーマを作成。画像と動画を保持しながら、テキストと背景を変換。 |
| **アニメーションを無効化** | CSSアニメーションとトランジションを軽減または削除。アクセシビリティと集中に役立ちます。 |

#### 上級（一部のサイトで問題が発生する可能性あり）

| Tweak | 機能 |
|-------|------|
| **可視スクロールバーを強制** | スクロールバーを自動非表示ではなく常に表示。一部のレイアウトと競合する可能性があります。 |
| **window.close()をブロック** | ウェブサイトがJavaScriptでタブやウィンドウを閉じるのを防止。 |

### Tweaksを有効にするには？

1. **Tweaks**タブに移動
2. 利用可能なオプションを閲覧
3. 任意のTweakをオンまたはオフに切り替え
4. 変更は即座に適用されます

### 特定のサイトでTweakが動作しないのはなぜ？

一部のウェブサイトは変更に対して積極的に対抗しています。Tweakが動作しない場合：

- サイトが積極的な変更防止技術を使用している可能性がある
- 競合を排除するために他の拡張機能を無効にしてみる
- GitHubで問題を報告してください。常に互換性を改善しています。

### 「メディアコントロールを強制」と「自動再生をブロック」の違いは？

- **メディアコントロールを強制**：サイトが非表示にしていても、標準のブラウザ動画/音声コントロールを表示
- **自動再生をブロック**：ページを読み込んだときに動画/音声が自動的に再生されるのを防止

両方を一緒に使用して、最大限のコントロールを得ることができます。

## アーカイブ検索

### アーカイブ検索とは？

ウェブは永久的ではありません。ページは削除され、記事は編集され、ウェブサイト全体が消えることもあります。Internet ArchiveのWayback Machineなどのサービスは、ウェブのスナップショットを保存し、特定の時点で何が存在していたかの歴史的記録を作成しています。

アーカイブ検索は、これらの保存されたバージョンへの即時アクセスを提供します：

- 削除されたコンテンツを見つけたり、変更前のページの様子を確認
- サイトが一時的にダウンしているときにページにアクセス
- ウェブサイトとその主張が時間とともにどのように進化したかを調査
- 改ざんされた可能性のある引用や出典を確認

### アーカイブを検索するには？

**Safariから：**
1. **AA**アイコンまたは**機能拡張**ボタンをタップ
2. **Trackless Links**を選択
3. 使用したいアーカイブサービスをタップ（例：Wayback Machine）
4. 現在のページがそのアーカイブで検索されます

**任意のアプリから（共有シート）：**
1. 任意のアプリからリンクを共有
2. **アーカイブで検索**を選択
3. 好みのアーカイブサービスを選択

### アーカイブサービスを追加できる？

はい。カスタムアーカイブサービスを追加できます：

1. **Tweaks**、次に**アーカイブ検索**、次に**サービスを設定**に移動
2. **+**をタップしてサービスを追加
3. アーカイブサービスのURLパターンを入力
4. サービスがポップアップで利用可能になります

### 「クエリパラメータを削除」を有効にすべき？

推奨：はい。このオプションはアーカイブ検索前にトラッキングコードやその他のパラメータを削除し、多くの場合より良いマッチにつながります。特定のパラメータを含むURLを検索する必要がある場合のみ無効にしてください。

## 共有シート連携

### 共有シートでリンクをクリーニングするには？

他のアプリからリンクを共有すると、トラッキングパラメータも一緒についてきます。共有シート拡張機能を使えば、共有する前にリンクをクリーニングできます：

1. 任意のアプリ（Safari、Reddit、Xなど）で**共有**ボタンをタップ
2. スクロールして**Trackless Linksでクリーニング**を見つける
3. クリーニングされたURLがクリップボードにコピーされます

これで、どこで見つけたかを明かさず、クリックした人全員を追跡しないクリーンなリンクを共有できます。

### 他のアプリからアーカイブに共有できる？

はい。同じ共有シートを使用：

1. 任意のリンクで**共有**をタップ
2. **アーカイブで検索**を選択
3. アーカイブサービスを選択
4. Safariがアーカイブ検索結果で開きます

## iCloud同期

### iCloud同期はどのように機能する？

プライバシー設定はあなたとともに移動します。iCloud同期を有効にすると、iPhone、iPad、Macで設定が個人のiCloudアカウントを通じて自動的に同期されます：

- **フィルター**（トラッキングパラメータ）
- **リダイレクト**（カスタムルールと有効状態）
- **Tweaks**（すべての設定）
- **アーカイブサービス**（設定したサービス）

プライバシー統計（ブロックされたトラッカー数など）はデバイス固有のままで、同期されません。

### iCloud同期を有効にする必要がある？

オプションです。iCloud同期はデフォルトで無効になっています。有効にするには：

1. アプリで**設定**に移動
2. **iCloud同期**を見つける
3. **オン**に切り替える

同じiCloudアカウントにサインインしているすべてのデバイスで設定が同期されます。

### デバイス間で変更が同期されない。何が問題？

以下を確認：

- 両方のデバイスでiCloud同期が**有効**になっている？
- 両方のデバイスが**同じiCloudアカウント**にサインインしている？
- **システム設定**（Mac）または**設定**（iOS）でiCloud Driveが有効になっている？
- 両方のデバイスがインターネットに接続されている？
- アプリを強制終了して再度開いてみる

それでも同期が機能しない場合は、iCloud同期をオフにしてから再度オンにしてみてください。

### iCloudなしでTrackless Linksを使える？

はい。iCloud同期は完全にオプションです。アプリは各デバイスにローカルで設定を保存しても問題なく動作します。

## ヒントとコツ

### どのトラッキングパラメータが削除されたか確認するには？

メインダッシュボードの**統計**をチェック。表示される内容：
- クリーニングされたURL総数
- 適用されたリダイレクト

### 設定をエクスポートできる？

設定はiCloud（有効な場合）またはデバイスにローカルで保存されます。すべてのデータはあなたのコントロール下にあり、サードパーティのサーバーに送信されることはありません。

### Trackless Linksはブラウジングを遅くする？

いいえ。拡張機能は非常に軽量で、ページの読み込みが完了する前にミリ秒単位でURLを処理します。パフォーマンスへの影響は感じられません。

### Trackless Linksを他のSafari拡張機能と一緒に使える？

はい。Trackless Linksは他の拡張機能と連携するように設計されています。競合が発生した場合は、Safariの拡張機能設定で実行順序を調整してみてください。

### Trackless LinksとTrackless Links Proの違いは？

| 機能 | Trackless Links | Trackless Links Pro |
|------|----------------|---------------------|
| iPhoneとiPad | はい | はい |
| Macアプリ | いいえ | はい |
| iCloud同期 | はい | はい |
| 価格 | 買い切り（iOSのみ） | ユニバーサル購入（全プラットフォーム） |

Trackless Links Proはユニバーサル購入です。一度購入すれば、iPhone、iPad、Macで使用できます。

## トラブルシューティング

### Safariで拡張機能が見つからない

確認事項：
- App Storeからアプリをインストールした
- アプリを少なくとも一度開いた
- システム設定でSafari拡張機能が有効になっている

### リンクがクリーニングされていない

確認事項：
- Safariで拡張機能が**有効**になっている？
- 必要な権限を許可した？
- フィルタータブでフィルターが**有効**になっている？

### カスタムリダイレクトが動作しない

- パターン構文を再確認（まず完全一致でテスト）
- ルールが**有効**になっていることを確認
- 他のルールが先にマッチしていないか確認（ルールの順序が重要）
